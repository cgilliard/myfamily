43 // 4.) Incomplete types
44 // Incomplete types are types that have one or more incomplete methods.
45 // Incomplete methods are methods that are not implemented for the defined type.
46 // They must be 'completed' by other Types that implement the 'incomplete' Type.
47 // This concept makes incomplete types similar to interfaces or traits in other
48 // programming languages. Incomplete types may NOT have fields. 'incomplete'
49 // types are denoted with the '@' symbol. This indicates that the type is
50 // 'incomplete'.
51 @MyIncompleteType
52 {
53  // This method which is also marked with the '@' must be implemented by
54  // the completing type. Note that there is an implicit 'self' variable
55  // that is not listed here but will be the first parameter passed to
56  // this method.
57  @my_required_method()->i32;
58  // This method which is NOT marked with the '@' provides a default
59  // which is implemented in the corresponding C file for the incomplete
60  // type (MyIncompleteType). The parameter list follows C conventions.
61  // This method also has an implicit self parameter.
62  my_default_implemented_method(u32 value)->i64;
63 }

66 // methods of the types it completes. In this case, 'my_required_method'. It can
67 // optionally override the default method 'my_default_implemented_method'. All
68 // of this is done in MyCompleteType's .c file.
69 MyCompleteType
70 {
71  i32 my_data;
72  i64 my_other_data;
73  // In 'complete' (or concrete) types, we use the '$' to indicate which
74  // fields are instance fields as opposed to type global. Similarly for
75  // methods, we do the same. This method is an instance method.
76  i32 my_method() -> i32;
